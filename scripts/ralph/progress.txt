## Codebase Patterns
- Tailwind CSS v4 uses `@import "tailwindcss"` in globals.css (not `@tailwind` directives)
- PostCSS config uses `@tailwindcss/postcss` plugin (not `tailwindcss` directly)
- next.config.ts uses `output: "export"` for static site generation to `out/` directory
- App Router structure: `app/layout.tsx` (root layout), `app/page.tsx` (home page), `app/globals.css` (global styles)
- CSS custom properties defined in `:root` in globals.css; Tailwind theme extends them via `var(--name)` in tailwind.config.ts
- Design tokens: --background (#0a0a0a), --foreground (#ededed), --muted (#737373), --border (#1f1f1f), --card (#111111), --accent (#e8d5c4)
- Fonts loaded via `next/font/google` with `variable` option; CSS classes applied to `<html>`; referenced via `var()` in --font-sans/--font-mono fallback chains
- In Next.js 14+, viewport settings use a separate `export const viewport: Viewport` (not inside `metadata`)
- Components live in `app/components/` with named exports; Container/Header/Footer are the shell components
- Content data files live in `content/` as JSON arrays; utility directories in `lib/`; static assets in `public/images/`
