## Codebase Patterns
- Sub-pages use `generateMetadata` with `getTranslations({ locale, namespace })` for locale-aware titles (template in layout adds ` | Name` suffix)
- Translation files are at `i18n/messages/{locale}.json` with namespaces like `site`, `nav`, `about`, etc.
- The archive page has no dedicated translation namespace — use `nav.archive` for its heading
- `params` in Next.js 15 App Router is a `Promise<{ locale: string }>` that must be awaited
- When switching a `next/font/google` font to a Google Fonts `<link>` stylesheet, define the CSS variable manually in `globals.css` (e.g. `--font-ma-shan-zheng: "Ma Shan Zheng", cursive`)
- The `@next/next/no-page-custom-font` lint rule is a Pages Router rule — in App Router, `<link>` tags in `<head>` for external fonts trigger a warning but it's a false positive
- Metadata route files (`app/sitemap.ts`, `app/robots.ts`) need `export const dynamic = "force-static"` when using `output: "export"` in Next.js
- On dark backgrounds (#0a0a0a), `text-neutral-600` (#525252) fails WCAG AA contrast (3.29:1) — use `text-neutral-500` (#737373, 4.97:1) as minimum
- SVG-only links (e.g. AnimatedSignature in header/footer) need `aria-label` on the wrapping `<Link>` or `<a>` tag
- Decorative elements (e.g. scroll arrows) should have `aria-hidden="true"` to avoid screen reader noise
