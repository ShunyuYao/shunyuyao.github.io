## Codebase Patterns
- Header component (`app/components/header.tsx`) accepts optional `pathname` prop; falls back to `usePathname()` hook
- All components in `app/components/` use **named exports** except AnimatedSignature, TypewriterTitle, LanguageSwitch, FluidCursorWrapper, FluidCursor
- Root layout (`app/layout.tsx`) is minimal (fonts, FluidCursor only); homepage uses `(home)` route group with its own layout for Container/Header/Footer; sub-pages use `PageLayout` directly
- Sub-pages (not in a route group) must use `PageLayout` which provides Header/Footer/sizing
- When moving pages into route groups, update relative imports to use `@/` alias
- Design tokens are CSS custom properties in `:root` (globals.css), referenced via `var(--name)` in Tailwind classes
- Content data lives in `content/` as JSON files: work.json, papers.json, awards.json, press.json, now.json
- Sub-pages are Server Components by default; do NOT use event handlers (onClick etc.) in them — restructure to avoid nesting interactive elements
- Dynamic routes (`[id]`) require `generateStaticParams` for static export; `params` is a `Promise` in Next.js 15 — must `await params`
- Paper/PaperLink types are defined in `PapersSection.tsx` and reused in `papers/page.tsx` — keep consistent if schema changes
- For grouping items by a field (year, type, etc.): use `reduce` to build a Record, then `Object.keys().map(Number).sort()` for ordered groups
- Stale `.next/` cache can cause cryptic build failures — delete it and rebuild if you see "Cannot find module" errors
